# 02-h1: The h^1 (h-max) Heuristic

**Problem ID:** Programming B -- h^1

## Problem Statement

Read an SAS+ file from standard input and output the value of the h^1 heuristic of the initial state. The value is guaranteed to be finite.

## Background

h^1 (also called h-max) is a delete-relaxation heuristic used in automated planning. It estimates the cost of reaching the goal from a given state by computing the cost of achieving each individual fact (variable=value pair) independently, then taking the **maximum** over all goal facts.

The key idea behind delete relaxation is that we ignore the negative effects of actions -- once a fact becomes true, it stays true forever. Under this simplification, the cost of achieving a set of facts simultaneously equals the cost of achieving the single hardest fact in that set.

## Algorithm

h^1 is computed via fixed-point iteration over a cost table indexed by facts.

### Definitions

- A **fact** is a pair (variable, value).
- The **preconditions** of an operator effect include the operator's prevail conditions, the effect's pre-value (if not -1), and the effect's conditional effect conditions.
- **cost[var][val]** stores the estimated cost of making variable `var` take value `val`.

### Procedure

1. **Initialize** the cost table:
   - `cost[var][val] = 0` if `state[var] == val` (fact is already true)
   - `cost[var][val] = infinity` otherwise

2. **Iterate** until no changes occur:
   - For each operator `o` and each of its effects `e`:
     - Compute `precond_cost = max { cost[v][d] : (v, d) is a precondition of this effect }`
     - If `precond_cost` is not infinity:
       - `new_cost = cost(o) + precond_cost`
       - If `new_cost < cost[e.variable][e.post_value]`, update and mark changed

3. **Extract** the heuristic value:
   - `h^1 = max { cost[var][val] : (var, val) is a goal fact }`

### Why max?

The h^1 heuristic uses max at two levels:
- **Precondition aggregation**: the cost of an operator is its action cost plus the cost of its most expensive precondition (since under delete relaxation, all cheaper preconditions are achieved "for free" along the way).
- **Goal aggregation**: the heuristic value is the cost of the single most expensive goal fact (again, all cheaper goals come for free).

This makes h^1 an **admissible** heuristic -- it never overestimates the true cost.

## Properties

- **Admissible**: h^1(s) <= h*(s) for all states s
- **Consistent**: h^1 satisfies the triangle inequality
- **Polynomial**: fixed-point converges in O(|facts|) iterations, each scanning all operators
- **Informative**: generally more informative than h^0 (goal-counting) but less than h^2

## Comparison with h^add (h^1 vs h-add)

h^1 uses `max` for goal aggregation, while h^add uses `sum`. This means:
- h^1 is admissible but often underestimates severely (only tracks the hardest subgoal)
- h^add is inadmissible but typically more informative in practice
